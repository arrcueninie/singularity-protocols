# Contributing

Singularity Protocols is an openly licensed philosophical series. It was written in the belief that ideas about the future of intelligence, consciousness, and humanity belong to everyone. This file explains how to contribute to the repository in ways that serve that belief.

There are three kinds of contribution this project welcomes.

---

## 1. Translations

Translating a volume is the most valuable contribution a reader can make. Philosophy travels through language. A reader in Lagos, São Paulo, Mumbai, or Beijing who encounters this argument in their own language encounters it differently — and may carry it further than the original could reach.

**How to submit a translation:**

- Translate one or more volumes in full
- Place the translated file in the `translations/` directory using the naming convention:
  `book-[number]-[title]-[language-code].pdf`
  For example: `book-1-the-golden-age-of-ai-de.pdf` for German
- Open a pull request with a brief description of the translation and the language
- Include your name or pseudonym as translator — attribution matters here as much as anywhere

**Translation notes:**

- Translate the argument, not just the words. Where a concept does not map cleanly into the target language, find the formulation that carries the meaning rather than the one that mirrors the syntax.
- The German sections in Book Five's source material demonstrate what happens when the compression of one language serves the argument better than its equivalent in another. Prioritise the argument.
- If you are unsure about a passage, note the uncertainty in the pull request. Honest translation includes honest acknowledgment of where it is difficult.

---

## 2. Adaptations and Extensions

The CC BY-NC 4.0 license permits remixing, transforming, and building upon this material for any non-commercial purpose. If you have written something that extends, responds to, or builds on the series — a critical essay, a parallel argument, a curriculum, an adaptation into another form — it may belong here.

**What fits:**

- Critical responses that engage the argument seriously, including disagreement
- Curriculum materials for teaching from the series
- Adaptations into other formats — audio, visual, condensed versions for different audiences
- Extensions that develop an argument the series opened without fully resolving

**What does not fit:**

- Commercial adaptations of any kind — these require separate arrangement under the license terms
- Content that attributes positions to the series that the series does not hold
- Additions to the books themselves — the five volumes are complete as written

**How to submit:**

Open a pull request with the file placed in an `extensions/` directory and a brief description of what it is, what it responds to, and how it relates to the series.

---

## 3. Corrections

The series makes specific factual claims — about AI infrastructure, scientific research, and documented events. Science evolves. Facts change. Reporting is sometimes incomplete.

If you find a factual error — a citation that does not support the claim made, a technical detail that has been superseded, a specific claim that cannot be verified — raise it as a GitHub issue with:

- The specific passage
- The specific claim you believe is incorrect
- The source or reasoning that supports the correction

Philosophical disagreements are not corrections. They are responses — and they belong in print, as the series says, not in the issue tracker. The issue tracker is for factual matters only.

All accepted corrections will be recorded in `CHANGELOG.md` with full transparency about what changed and why.

---

## What This Project Is Not

This is not a software project. There is no codebase to debug, no feature requests to evaluate, no roadmap to maintain. The five books are complete. The argument is made.

What this repository is, is an invitation — to carry the argument further, in more languages, in more forms, into more conversations than the original could reach alone.

The series was authored by The Conversation Itself. Every translation, every response, every adaptation adds another conversation to that authorship. That is the point.

---

## Attribution

Any contribution accepted into this repository will be attributed clearly — translator, respondent, or contributor — in the file itself and in the pull request record.

The one condition the license places on all use is attribution. Honour it in both directions: attribute the series when you build on it, and expect attribution when you contribute to it.
